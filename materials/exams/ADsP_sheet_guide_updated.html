1. 데이터의 특성(존재적 특성, 단위적 특성)을 안다.목표 1
존재적 특성당위적 특성객관적 사실추론의 근거

개념 정리

데이터는 두 가지 특성을 함께 지닙니다. 존재적 특성은 데이터가 '있는 그대로의 객관적 사실(fact)'이라는 점이고, 당위적 특성은 그 사실이 '추론·예측·전망을 위한 근거(raw material)'로 쓰인다는 점입니다.

실전 예시

기온이 23도라는 값 자체는 존재적 특성이고, 이 값을 근거로 '내일 냉방기 수요가 늘 것'이라 추론하면 당위적 특성이 작동한 것입니다.

시험에서 이렇게 나온다

'객관적 사실 자체'는 존재적 특성, '추론·예측의 근거로 쓰인다'는 표현은 당위적 특성으로 연결하는 짝짓기 문제가 자주 나옵니다.

관련 핵심어존재적 특성당위적 특성사실추론

핵심 암기 포인트

  • 존재적 특성 = 있는 그대로의 사실
  • 당위적 특성 = 추론·예측의 근거

2. 정형 데이터와 비정형 데이터를 설명할 수 있다.목표 2
정형데이터반정형데이터비정형데이터

개념 정리

정형 데이터는 행과 열처럼 고정된 스키마(구조)를 가진 데이터(RDB, 엑셀 표)입니다. 반정형 데이터는 스키마는 있지만 형태가 유동적인 데이터(XML, JSON, HTML, 센서 로그)이며, 비정형 데이터는 정해진 구조가 거의 없는 데이터(텍스트, 이미지, 영상, 음성)입니다.

실전 예시

고객 주문 테이블은 정형, 웹 API 응답 JSON은 반정형, 고객 상담 녹취 음성 파일은 비정형 데이터입니다.

시험에서 이렇게 나온다

시험에서는 정형/반정형/비정형 3분류로 확장해서 묻는 경우가 많습니다. 스키마의 유무와 고정 여부로 구분하세요.

관련 핵심어정형데이터반정형데이터비정형데이터RDBXML/JSON

핵심 암기 포인트

  • 정형 = 고정 스키마(RDB)
  • 반정형 = 스키마는 있으나 가변(XML/JSON)
  • 비정형 = 스키마 없음(텍스트·이미지·영상)

7. SQL(RDB)와 NoSQL에 대해 설명할 수 있다.목표 7
RDBNoSQL스키마

개념 정리

RDB(관계형 데이터베이스)는 행과 열로 구성된 고정 스키마를 가지며 정형 데이터 처리에 특화되어 있습니다(Oracle, MySQL, MariaDB). NoSQL(Not only SQL)은 고정된 스키마 없이 유연한 구조로 대용량의 비정형·반정형 데이터를 빠르게 처리합니다(MongoDB, Cassandra).

8. SQL의 네 분류와, 그에 속하는 명령어를 말할 수 있다.목표 8
DDLDMLDCLTCL

개념 정리

DDL(데이터정의어: CREATE, ALTER, DROP, RENAME)은 객체 구조를 정의하고, DML(데이터조작어: SELECT, INSERT, UPDATE, DELETE)은 데이터를 다루며, DCL(데이터제어어: GRANT, REVOKE)은 권한을 제어하고, TCL(트랜잭션제어어: COMMIT, ROLLBACK, SAVEPOINT)은 트랜잭션을 관리합니다.

실전 예시

테이블을 새로 만들 때는 CREATE(DDL), 데이터를 조회할 때는 SELECT(DML), 권한을 부여할 때는 GRANT(DCL)를 사용합니다.

각 명령어가 어느 분류에 속하는지 묻는 문제가 매우 자주 나옵니다. 명령어별 소속 분류를 정확히 암기하세요.

9. 빅데이터로 인한 변화를 설명할 수 있다.목표 9

10. 빅데이터의 특징 3V와 4V를 설명할 수 있다.목표 10
VolumeVelocityVarietyValueVeracity

개념 정리

기본 3V는 Volume(규모), Velocity(속도), Variety(다양성)입니다. 여기에 Value(가치)를 더하면 4V, Veracity(정확성·신뢰성)까지 더하면 5V로 확장됩니다.

개념 정리

지지도신뢰도향상도

개념 정리

연관분석(장바구니 분석)에서 지지도(Support)=P(A∩B)는 전체 거래 중 A와 B를 동시에 포함하는 거래의 비율이고, 신뢰도(Confidence)=P(A∩B)/P(A)는 A를 포함한 거래 중 B도 포함하는 비율이며, 향상도(Lift)=P(A∩B)/[P(A)×P(B)]는 A와 B가 우연히 같이 나타날 확률 대비 실제 함께 나타나는 정도로, 1보다 크면 양의 연관성, 1이면 서로 독립, 1보다 작으면 음의 연관성을 의미합니다.

실전 예시

전체 거래 100건 중 커피와 빵을 함께 산 거래가 10건이면 지지도 0.1, 커피 구매 40건 중 빵도 산 경우가 10건이면 신뢰도 0.25이며, 빵의 전체 구매비율이 0.2라면 향상도는 0.25/0.2=1.25로 양의 연관성을 보입니다.

시험에서 이렇게 나온다

세 지표의 공식(분자·분모)을 직접 대입해 계산하는 문제가 매우 자주 나오니, 지지도·신뢰도·향상도 공식을 정확히 암기하고 손으로 계산하는 연습을 하세요.

관련 핵심어지지도신뢰도향상도연관규칙

핵심 암기 포인트

  • 지지도 = P(A∩B)
  • 신뢰도 = P(A∩B)/P(A)
  • 향상도 = P(A∩B)/[P(A)P(B)]
  • 향상도>1: 양의 연관

DIKW 피라미드

Data → Information → Knowledge → Wisdom 순으로 가치가 높아집니다. Data는 가공되지 않은 사실, Wisdom은 지식이 축적되어 나온 창의적 판단입니다.

p-value 판단 규칙

p-value가 유의수준(α)보다 작으면 귀무가설을 기각합니다. 즉, p-value < α → '유의미한 차이가 있다'고 결론 내립니다.

정밀도 vs 재현율

정밀도(Precision)는 양성으로 예측한 것 중 실제 양성 비율(TP/(TP+FP)), 재현율(Recall·민감도)은 실제 양성 중 맞힌 비율(TP/(TP+FN))입니다.

K-means의 K

K는 학습을 시작하기 전에 미리 정해야 하는 군집의 개수입니다. 계층적 군집분석과 달리 사전에 K를 지정해야 합니다.

빅데이터 3V

Volume(규모), Velocity(속도), Variety(다양성)입니다. Value(가치)·Veracity(신뢰성)를 더하면 4V·5V로 확장됩니다.

DDL vs DML

DDL은 CREATE·ALTER·DROP처럼 구조를 정의하는 명령어, DML은 SELECT·INSERT·UPDATE·DELETE처럼 데이터를 다루는 명령어입니다.

릿지 vs 라쏘

릿지(L2)는 계수를 0에 가깝게 축소만 하고, 라쏘(L1)는 일부 계수를 정확히 0으로 만들어 변수 선택 효과를 냅니다.

배깅 vs 부스팅

배깅은 부트스트랩 표본으로 여러 모델을 병렬 학습(투표/평균), 부스팅은 이전 모델의 오답에 가중치를 주며 순차 학습합니다.

지지도·신뢰도·향상도

지지도=P(A∩B), 신뢰도=P(A∩B)/P(A), 향상도=P(A∩B)/[P(A)P(B)]. 향상도가 1보다 크면 양의 연관성을 의미합니다.

의사결정나무 분리기준

이산형 목표변수에는 지니지수(CART)·카이제곱(CHAID)·엔트로피(C4.5), 연속형 목표변수에는 분산감소량(CART)을 사용합니다.

정형·반정형·비정형

정형데이터는 고정 스키마(RDB), 반정형데이터는 유동적 스키마(XML·JSON), 비정형데이터는 스키마가 없는 텍스트·이미지·영상입니다.

하향식 vs 상향식 접근

하향식은 문제탐색→문제정의→해결방안탐색→타당성검토 순으로 진행하며 문제가 명확할 때 사용하고, 상향식은 비지도학습으로 데이터에서 문제를 재정의해 나갑니다.
