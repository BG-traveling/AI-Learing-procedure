OpenCV HSV 색상 시각화 도구

np.array([0, 100, 100])처럼 숫자로만 보이던 HSV 값을 실제 색상으로 바로 확인하는 HTML입니다. OpenCV 기준 HSV 범위인 H: 0~179, S: 0~255, V: 0~255를 그대로 사용합니다.

HSV → RGB 미리보기 lower / upper 범위 확인 색상 마스크 연습 빨간색 2구간 설명 OpenCV 코드 자동 생성

핵심 기억

H = Hue색상. OpenCV에서는 0~179만 사용합니다.
S = Saturation채도. 낮으면 회색에 가까워지고, 높으면 선명합니다.
V = Value밝기. 낮으면 어둡고, 높으면 밝습니다.

2. 자주 쓰는 색상 범위

카드를 누르면 해당 색상의 HSV lower/upper 값이 위 시각화 도구에 바로 적용됩니다. 실제 이미지마다 조명, 그림자, 카메라 설정이 달라서 S/V 값은 조정이 필요합니다.

빨간색은 왜 2개 범위가 필요할까?
OpenCV Hue는 원형 색상표를 0~179로 줄인 값입니다. 빨강은 시작점인 0 근처에도 있고 끝점인 179 근처에도 있습니다. 그래서 빨간색 마스크는 보통 [0,100,100]~[10,255,255]와 [170,100,100]~[179,255,255]를 따로 만든 뒤 합칩니다.

5. HSV 개념 정리

      

        

          
H: Hue, 색상

          

            색 자체를 의미합니다. 빨강, 노랑, 초록, 파랑처럼 “무슨 색인가?”를 나타냅니다.
            OpenCV에서는 일반적인 0~360도 색상환을 절반으로 줄여 0~179로 표현합니다.
          

        

        

          
S: Saturation, 채도

          

            색이 얼마나 선명한지입니다. S가 낮으면 회색/흰색에 가까워지고,
            S가 높으면 색이 선명해집니다. 범위는 0~255입니다.
          

        

        

          
V: Value, 밝기

          

            색이 얼마나 밝은지입니다. V가 낮으면 검정에 가까워지고,
            V가 높으면 밝아집니다. 범위는 0~255입니다.
          

        

      


      

        
RGB 말고 HSV를 쓰는 이유

        

          RGB는 빨강, 초록, 파랑 값을 직접 비교하기 때문에 조명 변화에 영향을 많이 받습니다.
          반면 HSV는 색상(H), 채도(S), 밝기(V)를 분리해서 생각할 수 있어서
          “빨간색 물체만 찾기”, “파란색 영역만 추출하기” 같은 색 기반 분할에 더 편합니다.
