1. 이미지 입출력과 NumPy 배열

이미지는 숫자 배열

cv2.imread() 결과는 NumPy 배열입니다. 컬러 shape는 (세로, 가로, 채널), 일반 dtype은 0~255를 담는 uint8입니다.

BGR과 RGB

OpenCV는 BGR, matplotlib은 RGB 순서입니다. 색이 이상하면 COLOR_BGR2RGB 변환을 먼저 확인합니다.

Gray와 저장

Gray는 채널이 없는 2차원 밝기 배열입니다. imwrite() 반환값이 True인지 확인해야 실제 저장 성공을 알 수 있습니다.

원본 보존

배열에 직접 선이나 도형을 그리면 원본이 바뀝니다. 결과 표시 전 original=image.copy()를 사용합니다.

import cv2
image = cv2.imread("sample.jpg")
if image is None:
    raise FileNotFoundError("경로와 파일명을 확인하세요")
print(image.ndim, image.shape, image.dtype)
rgb = cv2.cvtColor(image, cv2.COLOR_BGR2RGB)
gray = cv2.cvtColor(image, cv2.COLOR_BGR2GRAY)
ok = cv2.imwrite("gray.jpg", gray)
순서 함정: shape은 (세로, 가로, 채널)이지만 resize의 dsize는 (가로, 세로)입니다.

2. 밝기·대비·논리연산·히스토그램

포화 연산

cv2.add/subtract는 0~255를 넘는 값을 안전하게 제한합니다. uint8에 일반 NumPy 연산을 하면 값이 순환할 수 있습니다.

AND·OR·XOR·NOT

AND는 겹친 영역, OR은 합집합, XOR은 겹치지 않은 영역, NOT은 픽셀을 반전합니다. 마스크 적용의 기초입니다.

채널 분리

b,g,r=cv2.split(image)의 각 결과는 해당 색 성분 강도를 나타내는 2차원 배열입니다.

히스토그램

0~255 밝기별 픽셀 개수를 셉니다. 한쪽에 몰린 분포는 equalizeHist로 대비를 넓힐 수 있습니다.

bright = cv2.add(image, 100)
dark = cv2.subtract(image, 100)
hist = cv2.calcHist([gray], [0], None, [256], [0, 256])
equalized = cv2.equalizeHist(gray)

평활화 주의: 기본 equalizeHist()는 단일 채널 Gray 이미지에 적용합니다. 컬러 대비는 밝기 채널만 분리하는 방법을 고려합니다.

3. HSV 색공간과 특정 색상 마스크

HSV는 색상(H), 채도(S), 밝기(V)를 분리하므로 조명 변화가 있는 색상 분할에서 RGB보다 범위를 조절하기 쉽습니다. OpenCV의 H는 0~180 범위입니다.

hsv = cv2.cvtColor(image, cv2.COLOR_BGR2HSV)
lower_blue = np.array([90, 50, 50])
upper_blue = np.array([130, 255, 255])
mask = cv2.inRange(hsv, lower_blue, upper_blue)
result = cv2.bitwise_and(image, image, mask=mask)

mask

범위 안은 255, 밖은 0인 단일 채널 선택 지도입니다.

result

mask가 255인 위치의 원본 픽셀만 남긴 컬러 영상입니다.

빨간색은 두 범위: Hue 원형 축의 시작과 끝에 걸치므로 대략 0~20과 170~180을 각각 마스크로 만든 뒤 OR로 합칩니다.

4. 블러·이진화·모폴로지

평균·Gaussian·Median

평균은 주변 평균, Gaussian은 가까운 픽셀에 큰 가중치, Median은 중앙값을 사용해 소금-후추 노이즈에 강합니다.

Threshold

기준보다 크면 255, 아니면 0으로 분리합니다. 조명이 불균일하면 주변별 기준을 쓰는 adaptiveThreshold가 유리합니다.

침식·팽창

침식은 흰 영역을 깎고 팽창은 부풉니다. kernel 크기와 iterations가 효과 강도를 결정합니다.

Opening·Closing

Opening=침식→팽창으로 작은 흰 노이즈 제거, Closing=팽창→침식으로 작은 구멍과 끊김 보정입니다.

gaussian = cv2.GaussianBlur(gray, (7, 7), 0)
adaptive = cv2.adaptiveThreshold(
    gray, 255, cv2.ADAPTIVE_THRESH_MEAN_C,
    cv2.THRESH_BINARY, 11, 5)
kernel = np.ones((3, 3), np.uint8)
opening = cv2.morphologyEx(adaptive, cv2.MORPH_OPEN, kernel)

5. 크기·회전·Affine·Perspective

Resize와 보간

resize(image,(width,height))를 사용합니다. INTER_NEAREST는 빠르지만 확대 시 계단 현상이 생깁니다.

배열 Crop

image[y1:y2,x1:x2]처럼 세로 좌표를 먼저 씁니다.

Affine

2×3 행렬로 이동·회전·크기·기울이기를 표현하며 직선과 평행성을 유지합니다.

Perspective

네 점 대응으로 비스듬한 문서나 카드를 정면 사각형으로 보정합니다.

h, w = image.shape[:2]
matrix = cv2.getRotationMatrix2D((w/2, h/2), 30, 1)
rotated = cv2.warpAffine(image, matrix, (w, h))
P = cv2.getPerspectiveTransform(src_points, dst_points)
front = cv2.warpPerspective(image, P, (out_width, out_height))

6. 이진화와 특징 검출

Sobel·Canny

Sobel은 x/y 밝기 변화량을 구합니다. Canny는 노이즈 제거, 그래디언트, 비최대 억제, 이중 임계값으로 얇고 안정적인 엣지를 만듭니다.

Harris Corner

여러 방향으로 밝기 변화가 큰 지점을 코너로 판단합니다. 입력을 float32로 바꾸고 반응값을 정규화합니다.

Hough Circle·Line

엣지 점을 파라미터 공간에 투표해 원과 직선을 찾습니다. threshold, 최소 거리·길이·간격이 검출 결과를 좌우합니다.

Contour

이진 영상에서 연결된 경계를 찾습니다. RETR_EXTERNAL/LIST/TREE는 계층 검색 방식, CHAIN_APPROX_SIMPLE은 저장 좌표를 줄입니다.

edges = cv2.Canny(gray, 50, 150)
harris = cv2.cornerHarris(np.float32(gray), 2, 3, 0.04)
lines = cv2.HoughLinesP(edges, 1, np.pi/180, 100,
                        minLineLength=10, maxLineGap=10)
contours, hierarchy = cv2.findContours(
    binary, cv2.RETR_LIST, cv2.CHAIN_APPROX_SIMPLE)
cv2.drawContours(original, contours, -1, (0,255,0), 2)
버전 차이: findContours()의 반환값 개수는 OpenCV 버전에 따라 다를 수 있으므로 설치 버전의 문서를 확인합니다.

7. OpenCV 윤곽선(Contour) 검출

윤곽선은 같은 색이나 밝기를 가진 물체의 경계를 따라 이어진 점들의 집합입니다. 물체 개수 세기, 불량품 크기 검사, 도형 인식, 문서·동전·세포 영역 측정 등에 활용합니다. findContours()에는 일반 컬러 사진이 아니라 검출할 물체가 흰색(255), 배경이 검은색(0)인 단일 채널 이진 영상을 넣는 것이 핵심입니다.

전체 흐름: 이미지 읽기 → Gray 변환 → 블러로 노이즈 완화 → 이진화 또는 Canny → 모폴로지로 경계 보정 → 윤곽선 검색 → 면적·형태 필터링 → 원본 복사본에 시각화

1) 가장 많이 사용하는 실전 코드

import cv2
import numpy as np

image = cv2.imread("objects.jpg")
if image is None:
    raise FileNotFoundError("이미지 경로를 확인하세요.")

# ① 전처리: 윤곽선 입력은 단일 채널 이진 영상으로 만든다.
gray = cv2.cvtColor(image, cv2.COLOR_BGR2GRAY)
blurred = cv2.GaussianBlur(gray, (5, 5), 0)
_, binary = cv2.threshold(
    blurred, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)

# 흰 물체 안의 작은 구멍이나 끊어진 부분을 연결한다.
kernel = np.ones((3, 3), np.uint8)
binary = cv2.morphologyEx(binary, cv2.MORPH_CLOSE, kernel)

# ② 바깥쪽 윤곽선만 찾고, 직선 구간의 중복 좌표는 줄인다.
contours, hierarchy = cv2.findContours(
    binary, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)

# ③ 작은 노이즈를 제외하고 특징을 계산한다.
result = image.copy()
valid_count = 0
for contour in contours:
    area = cv2.contourArea(contour)
    if area < 500:
        continue

    valid_count += 1
    perimeter = cv2.arcLength(contour, True)
    x, y, w, h = cv2.boundingRect(contour)

    moments = cv2.moments(contour)
    if moments["m00"] != 0:
        cx = int(moments["m10"] / moments["m00"])
        cy = int(moments["m01"] / moments["m00"])
        cv2.circle(result, (cx, cy), 4, (0, 0, 255), -1)

    cv2.drawContours(result, [contour], -1, (0, 255, 0), 2)
    cv2.rectangle(result, (x, y), (x + w, y + h), (255, 0, 0), 2)
    cv2.putText(result, f"A:{area:.0f} P:{perimeter:.0f}",
                (x, max(y - 8, 15)), cv2.FONT_HERSHEY_SIMPLEX,
                0.45, (255, 255, 255), 1)

print("유효 물체 수:", valid_count)
cv2.imwrite("contour_result.jpg", result)

왜 Otsu 이진화인가?

히스토그램을 보고 임계값을 자동 결정합니다. 조명이 매우 불균일하면 adaptiveThreshold(), 이미 경계선 자체가 필요하면 Canny()를 비교해 봅니다.

물체가 검은색이라면?

THRESH_BINARY_INV로 반전합니다. 검은 물체를 그대로 넣으면 배경의 바깥 테두리를 물체로 잘못 찾기 쉽습니다.

왜 면적을 거르는가?

먼지와 잡음도 하나의 윤곽선이 됩니다. contourArea()의 최소·최대 범위를 정하면 관심 물체만 남길 수 있습니다.

왜 copy()를 쓰는가?

drawContours(), rectangle()은 전달한 배열을 직접 변경합니다. 원본과 전처리 결과를 비교하려면 복사본에 그립니다.

2) 검색 모드: 어떤 윤곽선을 가져올까?

RETR_EXTERNAL

가장 바깥 윤곽선만 반환합니다. 서로 떨어진 물체 수를 세거나 외형만 검사할 때 가장 단순하고 빠릅니다.

RETR_LIST

안쪽과 바깥쪽 윤곽선을 모두 반환하지만 부모·자식 관계를 구성하지 않습니다.

RETR_TREE

모든 윤곽선을 찾고 중첩된 부모·자식 관계를 완전한 트리로 기록합니다. 글자 구멍이나 도넛 구조 분석에 적합합니다.

RETR_CCOMP

바깥 경계와 내부 구멍을 2단계 계층으로 구성합니다. 채워진 영역과 구멍을 구분할 때 유용합니다.

hierarchy[0][i]의 네 값은 차례로 [다음 윤곽선, 이전 윤곽선, 첫 번째 자식, 부모]의 인덱스이며, 해당 관계가 없으면 -1입니다.

3) 좌표 저장 방식과 주요 특징값

CHAIN_APPROX_SIMPLE

직선 위의 중간 점을 생략하고 꼭 필요한 끝점 중심으로 저장합니다. 일반적인 검출에는 이 방식을 권장합니다.

CHAIN_APPROX_NONE

경계의 모든 점을 저장합니다. 세밀한 좌표가 꼭 필요한 경우에만 사용하며 메모리 사용량이 큽니다.

크기·위치

contourArea는 면적, arcLength(cnt, True)는 닫힌 경계의 둘레, boundingRect는 축에 평행한 사각형을 구합니다.

형태 분석

approxPolyDP는 다각형 단순화, convexHull은 볼록 껍질, minAreaRect는 회전된 최소 사각형을 구합니다.

# 꼭짓점을 단순화한다. epsilon이 클수록 점이 더 많이 줄어든다.
epsilon = 0.02 * cv2.arcLength(contour, True)
polygon = cv2.approxPolyDP(contour, epsilon, True)

# 예: 삼각형·사각형처럼 꼭짓점 수를 이용한 기초 분류
vertices = len(polygon)
if vertices == 3:
    shape_name = "triangle"
elif vertices == 4:
    shape_name = "quadrilateral"
else:
    shape_name = "circle or other"

4) 결과가 이상할 때 확인할 것

  • 윤곽선이 너무 많음: Gaussian/Median 블러, Opening, 최소 면적 필터를 차례로 적용합니다.
  • 하나의 물체가 여러 조각으로 나뉨: Closing 또는 팽창으로 끊어진 흰 영역을 연결합니다.
  • 서로 다른 물체가 하나로 합쳐짐: 팽창 강도를 낮추고, 이진화 임계값과 조명을 다시 조정합니다.
  • 안쪽 구멍을 찾지 못함: RETR_EXTERNAL 대신 RETR_TREE 또는 RETR_CCOMP를 사용합니다.
  • 중심점 계산 오류: 면적 모멘트 m00이 0인지 확인한 뒤 나눗셈을 수행합니다.
  • 좌표가 뒤바뀜: OpenCV 점은 (x, y), NumPy 슬라이싱은 array[y, x] 순서입니다.
스스로 확인하기: (1) 흰 물체가 아닌 영상을 넣으면 왜 문제가 생길까요? (2) 동전의 바깥 테두리만 셀 때 적절한 검색 모드는 무엇일까요? (3) 작은 점 잡음을 제거할 가장 간단한 두 방법은 무엇일까요?
정답: (1) 배경을 물체로 인식할 수 있어서, (2) RETR_EXTERNAL, (3) Opening과 최소 면적 필터입니다.
